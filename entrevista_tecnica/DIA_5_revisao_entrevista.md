## Exercícios (20)

### Revisão técnica relâmpago (1-9) — responda sem consultar, cronometre 2 min cada

1. [ ] Explique a diferença entre `==` e `===`.
2. [ ] Explique o que é uma closure com um exemplo.
3. [ ] Explique a diferença entre `map`, `filter` e `reduce`.
4. [ ] Explique a diferença entre PUT e PATCH.
5. [ ] Explique o ciclo RED → GREEN → REFACTOR do TDD.
6. [ ] Explique a diferença entre INNER JOIN e LEFT JOIN com um exemplo.
7. [ ] Explique como resolver um conflito de merge no Git.
8. [ ] Explique a diferença entre `props` e `state` no React.
9. [ ] Explique para que serve `useEffect` e cite um caso de uso com cleanup.

### Live coding rápido (10-12) — sem IDE, papel ou editor simples, 10 min cada

10. [ ] Resolva FizzBuzz de cabeça, explicando o raciocínio em voz alta enquanto escreve.
11. [ ] Escreva a assinatura de uma rota REST completa (`POST /livros`) explicando request/response/status codes possíveis, sem rodar o código.
12. [ ] Escreva uma query SQL que junte `livros` e `autores` e filtre por estoque baixo, de memória.

### Soft skills / comportamental (13-17) — usar técnica STAR (Situação, Tarefa, Ação, Resultado)

13. [ ] Prepare a resposta: "Me conta sobre um projeto que você fez — qual foi o maior desafio técnico?"
14. [ ] Prepare a resposta: "Já recebeu um feedback difícil? Como lidou?"
15. [ ] Prepare a resposta: "Como você se organiza quando tem várias tarefas com prioridades mudando?"
16. [ ] Prepare a resposta: "Descreva uma situação em que precisou aprender algo novo rapidamente."
17. [ ] Prepare a resposta: "Por que você quer trabalhar na Foxconn/nessa vaga?"

### Fechamento (18-20)

18. [ ] Prepare 2-3 perguntas para fazer ao entrevistador (ex: "Como é o processo de code review no time?", "Quais tecnologias do dia a dia além das citadas na vaga?").
19. [ ] Revise seu currículo/projetos em voz alta — para cada projeto listado, consiga explicar em 30 segundos o que foi feito e qual sua contribuição.
20. [ ] Teste o ambiente de live coding (Node instalado, editor configurado, terminal funcionando, conexão estável) — resolva um exercício qualquer do Dia 1 do zero para validar que está tudo pronto.
